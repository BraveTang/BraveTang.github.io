1.C++与STL速成

2017-03-22
C++

#C++基础知识

  • 在C中，被包含的头文件以.h结尾，C++中标准库中头文件省略了这后缀，C的标准库在C++仍然存在，但换了新名字。如变成了
  • 强制类型转换eg: int 3 = static_cast(myFoalt); 是C++中最复杂，最整洁的推荐方法。
  • 枚举类型是一个整数值；强类型枚举 enum class 是类型安全的。
  • C++ 中利用 std::array 代替数组优点：它总是知道自身大小，不会自动转换为指针，具有迭代器，方便变量元素。
  • C风格的数组及std::array 是固定固定大小的，编译的时候就固定下来了，要数组的大小是动态的，使用std:vector。
  • 基于区间的for循环允许方便的迭代容器中的元素，用于数组，初始化列表，具有返回的begin()和end()函数的类型。
  • 函数声明通常称为“函数原型”或“签名”，以强调这代表函数的访问方式，而不是具体代码。
  • 每个函数都有一个预定于的局部变量func,可以将这个用于日志。
  • 替代的函数语法 auto func(int i ) -> int {return i +2;} 函数的返回类型不再位于开头，而放到行尾->后，调用不变。
  • auto 用途：编译时自动判断变量类型；替代函数语法；函数返回类型推断；用于通用lambda表达式。
  • decltype 把表达式作为实参，计算出该表达式的类型。

#深入研究C++

  • C++中内存分为堆栈和堆。堆栈上分配内存的变量自动释放，堆是与当前函数或堆栈帧完全没有关系的内存去区域，如果想在函数调用结束仍然保留其中的声明的变量，可将变量放到堆中，程序可以任何时候向堆中添加或修改中已有的位，必须确保释放在堆上分配的任何内存，这个过程不会自动完成，除非使用智能指针。
  • 任何时候应避免使用位初始化的变量，尤其是未初始化的指针，它会指向内存中的每个随意位置，使程序崩溃。不希望分配内存，可初始化为空指针nullptr.
  • 使用完动态内存，用delete操作释放内存，为防止在释放指针指向的内存后再使用指针，建议把指针设置设置为nullptr.
  • 对null或为初识化的指针解除引用会导致程序崩溃。
  • 避免使用maclloc()和free()，而使用new和 delete,或 new[]和delete[],并且保证成对出现，避免内存泄露。
  • C++提供3钟智能能指针std::uique_ptr、std::shared_ptr、std::wedk_ptr。

    uiquie_ptr超出作用域时，会自动释放内存和资源。
    shared_ptr允许数据的分布式所有权，每次指定shared_ptr是，都递增引用计数，超出作用域时，就递减引用计数。当计数为0时就表示数据不再有任何拥有者，于是释放指针引用的对象。
    weak_ptr 可以观察 shared_ptr,而不会递增或递减锁连接的shered_ptr的引用计数

  • 利用decltype(auto) 来推断表达式的类型，就不需要引用限定符合和const限定符了，且解决代码重复的问题

#作为面向对象语言的C++

  • 推荐把所有数据类型声明为prviate,在需要时，通过public读取器和设置器来访问他们。
  • 为了遵循const-correctness原则，最好将不改变对象的任何数据成员的成员函数声明为const,相对于非const成员函数“修改器”，这些成员函数成为”检测器”。
  • 对象的创建可以基于堆栈方式，也可以基于堆的方式。
  • 尽量使用STL标准库的类来解决问题。

#第一个有用的C++程序

  • #pragma once 以防止文件被多次包含。
  • 当类中存在获取器时，使用获取器是一种好的风格，甚至在类的内部也是如此。
